# ADR Template

Use this template when a change is architecturally significant.

## Title

Short decision title.

## Status

Proposed / Accepted / Superseded

## Context

What problem, requirement, or constraint led to this decision?

## Decision

What was chosen?

## Rationale

Why is this choice better than the alternatives for this project right now?

## Consequences

What becomes easier, harder, riskier, or more constrained after this decision?

## References

List internal files, issues, and external documents that informed the decision.
